Referencia Técnica (Javadoc)

Documentación técnica de controladores y funciones del sistema Casafinder.

Versión: 1.0  |  Fecha: Febrero 2026

1. Controlador de Autenticación

Ubicación: backend/src/controllers/controladorAuth.js

registrar(req, res)

Registra un nuevo usuario en el sistema con validación y hashing de contraseña.

  • Parámetros:
    • req.body.nombre_completo: Nombre real del usuario.
    • req.body.correo_electronico: Email único en el sistema.
    • req.body.contrasena: Contraseña segura.
  • Retorno: JSON indicando éxito o errores de validación.

login(req, res)

Gestiona el inicio de sesión. Soporta flujo estándar y 2FA.

  • Lógica: Si el usuario tiene tfa_enabled, no devuelve token, sino un indicador require2fa.
  • Retorno Exitoso: { token: JWT, usuario: Object }

setup2FA(req, res)

Inicia el proceso de configuración de Doble Factor.

  • Lógica: Genera un secreto TOTP único y devuelve una URL para código QR.
  • Detalle: El emisor del QR se establece como "Casafinder (email)" para fácil identificación.

2. Controlador de Propiedades

Ubicación: backend/src/controllers/controladorPropiedades.js

obtenerPropiedades(req, res)

Recupera el listado público de inmuebles. Soporta filtrado.

crearPropiedad(req, res)

Da de alta una nueva propiedad vinculada al usuario autenticado.

  • Lógica: Si no se proporciona imagen, el sistema asigna una de stock aleatoria de Unsplash.

descargarPDF(req, res)

Genera y transmite un folleto PDF de la propiedad al vuelo.

  • Salida: Stream binario application/pdf.

3. Controlador de Mensajes

Ubicación: backend/src/controllers/controladorMensajes.js

crearConsulta(req, res)

Inicia un hilo de chat. Impide que un propietario se escriba a sí mismo.

4. Scripts Frontend

Ubicación: frontend/src/mensajes.js

loadMessages(id, backgroundUpdate)

Renderiza el chat activo.

  • Param: backgroundUpdate (Bool) - Si es true, silencia el indicador "Cargando..." para permitir actualizaciones en segundo plano sin parpadeos.
