Documentación Técnica Completa

Referencia exhaustiva del código fuente de Casafinder. Incluye definición de modelos, rutas de API REST, controladores y lógica de frontend.

1. Modelos de Datos (Sequelize)

Ubicación: backend/src/models/

MODEL Usuario

Representa a un actor del sistema (Administrador o Cliente).

Campo Tipo Descripción
nombre_completo STRING Nombre real del usuario.
correo_electronico STRING Único. Usado para login (user).
contrasena STRING Hash BCrypt. Nunca texto plano.
tfa_enabled BOOLEAN Si tiene 2FA activo (true/false).
secret_2fa STRING Clave secreta Base32 para TOTP.
token_confirmacion STRING UUID temporal para verificar email.
MODEL Propiedad

Inmueble publicado en la plataforma.

Campo Tipo Descripción
titulo STRING(150) Título comercial del anuncio.
descripcion TEXT Descripción detallada.
precio DECIMAL Precio de venta en Euros.
imagen STRING URL de la imagen principal.
publicado BOOLEAN Visibilidad pública.
userId INTEGER FK -> Propietario (Usuario).

2. Controlador de Autenticación

Archivo: backend/src/controllers/controladorAuth.js

POST registrar(req, res)

Orquesta el registro de nuevos usuarios.

Uso: /auth/registro

Lógica Interna:

  1. Valida `express-validator` para campos vacíos.
  2. Verifica si correo_electronico ya existe.
  3. Crea el usuario con el hook beforeCreate que hashea la contraseña.
  4. (Simulado) Envía email de bienvenida.
POST login(req, res)

Inicio de sesión seguro con soporte para flujo 2FA.

Retorno (Estándar):

{
  "token": "eyJhbGciOiJIUz...", 
  "usuario": { "id": 1, "nombre": "Juan" }
}

Retorno (Si tiene 2FA):

{
  "require2fa": true,
  "userId": 1,
  "msg": "Introduce código"
}
GET setup2FA(req, res)

Genera configuración inicial para Google Authenticator.

  • Genera secreto aleatorio Base32 (lib: speakeasy).
  • Genera QR Code DataURL (lib: qrcode).
  • El QR incluye el "Issuer" como Casafinder (email).
POST verify2FA(req, res)

Valida el código temporal de 6 dígitos.

Params: { userId, token }

Lógica: Usa speakeasy.totp.verify() con ventana de tiempo de 1 (30s margen).

3. Controlador de Propiedades

Archivo: backend/src/controllers/controladorPropiedades.js

POST crearPropiedad(req, res)

Alta de inmuebles con asignación automática de usuario y placeholder de imagen.

  • Fallback de Imagen: Si el frontend no envía imagen, el controlador selecciona aleatoriamente una URL de Unsplash predefinida.
  • Validación: Requiere campos como precio, habitaciones > 0.
GET obtenerPropiedades(req, res)

Motor de búsqueda pública.

  • Filtro `search`: Aplica Op.like en Título Y Descripción.
  • Include: Carga ansiosamente (Eager Loading) la Categoría y el Usuario propietario.
GET descargarPDF(req, res)

Generación de documentos dinámicos.

  • Librería: PDFKit.
  • Output: Stream Buffer directo al navegador.

4. Utilidades y Middleware

MW checkAuth

Protege rutas privadas.

  1. Busca header Authorization: Bearer TOKEN.
  2. Decodifica JWT con process.env.JWT_SECRET.
  3. Busca el usuario en DB.
  4. Si OK, inyecta req.usuario y llama a next().
HELPER emails.js

Gestión de comunicaciones SMTP (Mailtrap/Gmail).

  • emailRegistro: Envía token de verificación.
  • emailOlvidePassword: Envía enlace de reset.
  • Modo DEV: Si no hay credenciales SMTP, imprime el link en consola.

5. Scripts de Frontend

Ubicación: frontend/src/

JS api.js

Wrapper centralizado para fetch.

Inyecta automáticamente el Token JWT almacenado en localStorage en cada petición.

JS mensajes.js / mensajes-admin.js

Lógica de chat en tiempo real simulado.

  • Polling: setInterval cada 10s. Llama a loadMessages(id, true).
  • UX: La flag true evita refrescar el DOM con "Cargando...", eliminando parpadeos.
